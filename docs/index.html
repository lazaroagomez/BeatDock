QUICK SETUP

Get BeatDock running in just 5 simple steps

1

🔒 Enable Discord Intents

Required: Enable these three Privileged Gateway Intents in Discord Developer Portal

Presence Intent
Server Members Intent
Message Content Intent
Critical: The bot will not function without these intents enabled in your Discord Developer Portal under Bot → Privileged Gateway Intents.
2

Clone the Repository

Get the source code on your machine

git clone https://github.com/lazaroagomez/BeatDock.git
cd BeatDock
3

Configure Environment

Create a .env file with your bot credentials

# Discord Bot Configuration
TOKEN=your_discord_bot_token_here
CLIENT_ID=your_discord_client_id_here

# Optional: Spotify Configuration
# Get your credentials from: https://developer.spotify.com/dashboard/applications
SPOTIFY_ENABLED=false
SPOTIFY_CLIENT_ID=your_spotify_client_id_here
SPOTIFY_CLIENT_SECRET=your_spotify_client_secret_here

# Lavalink Configuration
LAVALINK_HOST=lavalink
LAVALINK_PORT=2333
LAVALINK_PASSWORD=youshallnotpass

# Optional Settings
DEFAULT_LANGUAGE=en
ALLOWED_ROLES=

# Optional: Audio Settings
DEFAULT_VOLUME=80
4

Use Official Docker Image

Replace the bot service's build section with the pre-built image:

# Complete docker-compose.yml configuration
services:
  bot:
    container_name: beatdock
    image: ghcr.io/lazaroagomez/beatdock:latest  # Use official image instead of build
    depends_on:
      - lavalink
    networks:
      - beatdock-network
    env_file: .env

  lavalink:
    container_name: beatdock-lavalink
    image: ghcr.io/lavalink-devs/lavalink:4-alpine
    ports:
      - "2333:2333"
    networks:
      - beatdock-network
    volumes:
      - ./application.yml:/opt/Lavalink/application.yml
    environment:
      - LAVALINK_PASSWORD=${LAVALINK_PASSWORD:-youshallnotpass}
      - SPOTIFY_ENABLED=${SPOTIFY_ENABLED:-false}
      - SPOTIFY_CLIENT_ID=${SPOTIFY_CLIENT_ID:-}
      - SPOTIFY_CLIENT_SECRET=${SPOTIFY_CLIENT_SECRET:-}

networks:
  beatdock-network:
    name: beatdock_network
5

Deploy Commands & Start

Register slash commands and launch BeatDock

docker compose run --rm bot npm run deploy
docker compose up -d

Build from Source

If you prefer to build the Docker image yourself

Alternative Setup

Keep the original build configuration in your docker-compose.yml instead of using the pre-built image.

services:
  bot:
    container_name: beatdock
    build:
      context: .
      dockerfile: Dockerfile

Then follow the same steps to deploy commands and start the bot.

Community ARM64 Image: Thanks to @driftywinds for providing an ARM64 community image at ghcr.io/driftywinds/beatdock-bot:latest. See Issue #32 for details.

PERMISSIONS

Control who can use your bot

Admin Override

Users with Administrator permissions always have full access to all bot commands

Role-Based Access

Configure specific roles that can use the bot by adding role IDs to your .env file

ALLOWED_ROLES=123456789012345678,234567890123456789

Default Behavior

If no roles are specified, everyone can use the bot
